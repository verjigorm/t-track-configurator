# T-Track Bolt Insert Configurator — Specification

## Overview

A static single-page web application that lets users configure a parametric T-track sliding bolt insert, preview it interactively in 3D, and download an STL file for manufacturing (3D printing or CNC machining).

No backend — all computation happens client-side.

## The Insert

A sliding insert that fits inside a T-track channel. It has:

- A **T-shaped cross-section** that matches the T-track profile (narrow slot on top, wider lip below).
- A **hex bolt head pocket** on the underside, capturing a standard hex bolt head.
- A **clearance hole** through the body for the bolt's threaded shaft, which protrudes upward through the top of the insert.
- **Single bolt** per insert.
- **0.5mm chamfers** on edges for easier insertion and better print quality.
- rectangular steps at the bottom of the bolt head cavity which enable printing without supports using bridging 
- an image/legend showing visually what each measurement means


The user slides the insert into the T-track from one end. A bolt is placed head-down into the pocket, and the threaded shaft sticks up through the track slot for fastening.

## Parameters

All dimensions are user-specified with sensible defaults. Units are switchable between millimeters and inches.

### T-Track Profile

| Parameter        | Description                              | Default (mm) |
| ---------------- | ---------------------------------------- | ------------ |
| Slot width       | Width of the narrow slot opening on top  | 19.0         |
| Lip width        | Width of the wider undercut below        | 30.0         |
| Slot depth       | Height of the T-track slot part          | 11.0         |
| Lip depth        | Height of the T-track lip part           | 11.0         |

### Bolt Dimensions

| Parameter        | Description                              | Default (mm) |
| ---------------- | ---------------------------------------- | ------------ |
| Head width       | Flat-to-flat width of the hex bolt head  | 10.0         |
| Head height      | Thickness of the hex bolt head           | 4.0          |
| Shaft diameter   | Diameter of the bolt's threaded shaft    | 6.0          |

### Insert Dimensions

| Parameter        | Description                              | Default (mm) |
| ---------------- | ---------------------------------------- | ------------ |
| Insert length    | Total length of the insert body          | 25.0         |
| Chamfer          | The chamfer on edges parallel to the track | 0.5        |

## User Interface

### Layout

- **Desktop**: Side-by-side — parameter inputs on the left, 3D preview on the right.
- **Mobile**: Stacked — parameters on top, 3D preview below.

### Controls

- Labeled input fields for each parameter, grouped by category (Track, Bolt, Insert).
- Unit toggle (mm / inches) that converts all displayed values.
- Render button triggering the update of the rendered onject
- "Generate STL" / "Download" button.

### 3D Preview

- Interactive orbit/zoom/pan viewer powered by **Three.js**.
- Renders the STL mesh generated by OpenSCAD WASM.
- Shows the insert in a neutral color with basic lighting.

## Technical Architecture

### Stack

- **Vanilla JS** (or lightweight framework) — no heavy dependencies.
- **OpenSCAD WASM** — parametric model generation and STL export, runs client-side.
- **Three.js** — interactive 3D rendering of the generated mesh.

### Pipeline

1. User adjusts parameters in the UI.
2. Parameters are passed to an OpenSCAD `.scad` model via the WASM interface.
3. OpenSCAD WASM compiles the model and outputs an STL (binary).
4. The STL is parsed and rendered in the Three.js viewer.
5. The same STL binary is available for download.

### File Structure

```
/src          — Application code (JS, CSS, HTML)
/scad         — OpenSCAD model files (.scad)
/docs         — GitHub Pages deploy target
```

### OpenSCAD Model

A single `.scad` file that takes all parameters as variables and produces the insert geometry. The model constructs:

1. The T-shaped body (rectangular prism with the lip profile).
2. The hex pocket (subtracted from the underside).
3. The clearance hole (subtracted through the full height).

## Deployment

- Hosted on **GitHub Pages**.
- Served from `/docs` folder (or `gh-pages` branch).
- No build step required — static files only.

## Target Users

- **3D printing hobbyists** — woodworkers and makers printing custom jigs and fixtures.
- **CNC/machining users** — users machining inserts from metal or plastic.

Both workflows consume the same STL output.
